Encapsulation

A mechanism of wrapping the data (variables) and code acting on the data (methods) together as a single unit. In encapsulation, the variables of a class will be hidden from other classes, and can be accessed only through the methods of their current class. Therefore, it is also known as data hiding.

Access Modifiers

Keywords in object-oriented languages that set the accessibility of classes, methods, and other members. Access modifiers are a specific part of programming language syntax used to facilitate the encapsulation of components.

  • Private - Visible only to objects of the classes in which it is defined.
    • If you do not specify an access modifier, the method or variable is private.
  • Protected - This is the default, and means that the method or variable is accessible only within the Apex class in which it is defined.
    • You can only use this access modifier for instance methods and member variables.
  • Public - The class, method, or member is accessible in any Apex code in this application or namespace.
  • Global - The method or variable can be used by any Apex code that has access to the class, not just the Apex code in the same application.
    • If used for a method or variable the class must also be global.
