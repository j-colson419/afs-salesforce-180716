Encapsulation

A mechanism of wrapping the data (variables) and code acting on the data (methods) together as a single unit. In encapsulation, the variables of a class will be hidden from other classes, and can be accessed only through the methods of their current class. Therefore, it is also known as data hiding.

Access Modifiers

Keywords in object-oriented languages that set the accessibility of classes, methods, and other members. Access modifiers are a specific part of programming language syntax used to facilitate the encapsulation of components.

  • Private - Visible only to objects of the classes in which it is defined.
    • If you do not specify an access modifier, the method or variable is private.
  • Protected - This is the default, and means that the method or variable is accessible only within the Apex class in which it is defined.
    • You can only use this access modifier for instance methods and member variables.
  • Public - The class, method, or member is accessible in any Apex code in this application or namespace.
  • Global - The method or variable can be used by any Apex code that has access to the class, not just the Apex code in the same application.
    • If used for a method or variable the class must also be global.

Inheritance
  • Use the keyword extends to inherit methods and variables from a class.
    • The parent class can be an abstract or non-abstract class
    • A class can only extend one class.
  • Use the keyword implements to inherit methods from a interface.
    • All methods must be instantiated.
    • A class can implement multiple interfaces.
  • Apex does not support multiple interitance.
Polymorphism

The ability of an object to take on many forms.

Overloading

Occurs when two or more methods in one class have the same method name but different parameters.

  • Some contest whether polymorphism applies to overloading.

Overriding

Having two methods with the same method name and parameters (i.e., method signature). One of the methods is in the parent class and the other is in the child class. Overriding allows a child class to provide a specific implementation of a method that is already provided its parent class.

Abstraction

Abstract Classes

  • Create an abstract class by putting the abstract modifier before the class keyword.
  • A class that contains at least one abstract method.
    • An abstract method is a method that has a signature, but not body.
  • Abstract classes can have concrete (not abstract) methods and variables.
  • When an abstract class is subclassed, the subclass usually provides implementations for all of the abstract methods in its parent class.
    • If it does not, then the subclass must also be declared abstract.

Interfaces

  • Create an interface by putting the interface keyword in place of the class keyword.
  • A class that contains only abstract methods.
  • Similar to an abstract class, when an interface is subclassed, the subclass usually provides implementations for all of the abstract methods in its parent interface.
Additional Notes

Control Flow Statements

A statement that controls the flow of your code.

If-Else

Expressions must evaluate to a boolean value.

Switch

Expressions can evaluate to any of the following:

  • Integer
  • Long
  • sObject
  • String
  • Enum

Loops

Allow you to run a block of code multiple times until loop completion. All loops allow for loop control structures:

  • break; exits the entire loop
  • continue; skips to the next iteration of the loop

Static vs. Instance

Static

Static methods, variables, and initialization code have these characteristics:

  • They’re associated with a class, not an instance of a class.
  • They’re allowed only in outer classes.
  • They’re initialized only when a class is loaded.
  • They aren’t transmitted as part of the view state for a Visualforce page.

Instance

Instance methods, member variables, and initialization code have these characteristics.

  • They’re associated with a particular object.
  • They have no definition modifier.
  • They’re created with every object instantiated from the class in which they’re declared.

The Sharing Modifier

The with sharing keyword allows you to specify that the sharing rules for the current user be taken into account for a class. You have to explicitly set this keyword for the class because Apex code runs in system context.

With Sharing

  • Allows you to specify that the sharing rules for the current user should be taken into account for the class.
  • with sharing is a keyword that can be used in class definitions.

Without Sharing

  • Ensures that the sharing rules for the current user are not enforced.
  • without sharing is also a keyword that can be used in class definitions.

Some things to note about using the sharing keyword:

  • The sharing setting of the class where the method is defined is applied, not of the class where the method is called. For example, if a method is defined in a class declared with with sharing is called by a class declared with without sharing, the method will execute with sharing rules enforced.
  • If a class isn’t declared as either with or without sharing, the current sharing rules remain in effect. This means that the class doesn’t enforce sharing rules except if it acquires sharing rules from another class. For example, if the class is called by another class that has sharing enforced, then sharing is enforced for the called class.
  • Both inner classes and outer classes can be declared as with sharing. The sharing setting applies to all code contained in the class, including initialization code, constructors, and methods.
  • Inner classes do not inherit the sharing setting from their container class.
  • Classes inherit this setting from a parent class when one class extends or implements another.

Constructor

Code that is invoked when an object is created.

  • Apex provides you by default a no parameter constructor for each class.
  • When you define your own constructor, you no longer have access to the default constructor.
  • Constructors can be overloaded, each having their own signatures.

Constructor Chaining

When multiple constructors fire on object creation.

  • You will only use this(...) in constructor chaining.
  • If you are going to user constructor chaining, the chaining must be the first statement in the constructor.

Super

  • Can be used by child classes.
  • Allows you to override constructors and methods from the parent class.

Inner Classes

  • A class within a class.
  • Default access modifier is private.
  • Don't inherit with/without sharing keywords.

Wrapper Classes

A wrapper (or container) class is a class, data structure, or an abstract data type whose instances are a collections of other objects.
