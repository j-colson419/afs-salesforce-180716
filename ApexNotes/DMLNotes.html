Data Manipulation

Database Manipulation Language (DML)

  • read operations - writen in SOQL
  • write operations
    • insert
    • update
    • delete
    • upsert
    • merge
    • undelete
  • Insert, update, and delete - if one fails, they all fail!

Database Classes

Actions performed on database classes behave the same as the DML operations for the following Database methods:

  • insert()
  • update()
  • delete()

For example: Database.insert(new Account(Name='N'), false); behaves the same as insert new Account(Name='N');

Important Notes

  • You can have a maximum of 150 DML operations per transaction.
  • You can have a maximum of 10,000 rows per DML operation.
  • Use the SaveResult class to get information about a successful or erroneous Database insert or update.
  • Use the DeleteResult class to get information about a successful or erroneous Database delete.
SOQL & SOSL

Salesforce Object Query Langueage (SOQL)

Evaluates to a list of sObjects, a single sObject, or an integer.

  • Used when we know which object the data resides in.
  • Retrieve data from single object or multiple related objects.
  • Count the number of records meeting specific criteria.
  • Sort results within the query.
  • Retrieve data from number, date, or checkbox fields.

Salesforce Object Search Langueage (SOSL)

Evaluates to a list of lists of sObjects.

  • Used when we don't know which object the data resides in.
  • Retrieve data for a specific term.
  • Retrive multiple non-related objects/fields.
  • Retrieve data in Chinese, Japanese, or Thai.

Where to write SOQl queries?

  • In Schema Explorer in Force.com IDE.
  • In Query Editor tab in DevCons.
  • In Apex Statements
  • In VisualForce controllers and getter methods.
  • In the queryString paramter in a query().

Where to write SOSl queries?

  • In Schema Explorer in Force.com IDE.
  • In Query Editor tab in DevCons.
  • In Apex Statements
  • In VisualForce controllers and getter methods.
  • In the queryString paramter in a search.query().
Additional Notes

Try-Catch Statements

Used to run code that might be erroneous and fail. If the code fails you can run other code to correct for the failure or make note of it.
